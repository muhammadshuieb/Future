Future Radius — FreeRADIUS 3 (Debian) + MySQL (external schema)
================================================================

Docker
------
  docker compose up -d mysql freeradius

Debug (verbose, foreground):
  FREERADIUS_DEBUG=1 docker compose run --rm --service-ports freeradius

Bare metal (example)
--------------------
  sudo ln -sf /etc/freeradius/3.0/mods-available/sql /etc/freeradius/3.0/mods-enabled/sql
  sudo systemctl restart freeradius
  sudo freeradius -X    # test without systemd

Active sessions (truth in MySQL)
--------------------------------
  FreeRADIUS does not expose a portable "online" state; this project and SaaS logic use radacct:

    acctstoptime IS NULL   → session still open (active)
    acctstoptime IS NOT NULL → session ended

  CoA, dashboards, and accounting use radacct-based queries.

radacct cleanup (keep ~30 days of closed sessions)
-------------------------------------------------
  - File: docker/mysql/opt/04-radacct-cleanup-event.sql
  - Daily EVENT: deletes rows where acctstoptime IS NOT NULL and older than ~30 days.
  - Requires event_scheduler=ON (enabled in docker/mysql/conf.d/radius-import.cnf).
  - Apply manually if needed: run the SQL file as a MySQL superuser.

MikroTik — Interim-Update (live usage)
--------------------------------------
  /radius set [find] interim-update=1m

  Or per server index:
  /radius set numbers=0 interim-update=1m

MikroTik — CoA / Disconnect (important)
---------------------------------------
  Without accepting incoming from the RADIUS server, Disconnect-Request on UDP 3799 will not work:

  /radius incoming
  set accept=yes

  (WinBox: IP → RADIUS → Incoming → Accept)

  Ensure the NAS secret in table `nas` matches `/radius` on the MikroTik.

Quick tests
-----------
  radtest USER PASSWORD RADIUS_HOST 0 SECRET
  radtest -x USER PASSWORD RADIUS_HOST 1812 SECRET

FreeRADIUS 3.2.x (Debian bookworm)
-----------------------------------
  - `group_attribute` must be defined in the sql module (see mods-available/sql.template);
    otherwise startup fails with: Reference "${group_attribute}" not found in queries.conf.

NAS IP (WireGuard / "unknown client")
-------------------------------------
  `nasname` should match the UDP 1812 source as seen by the server (not necessarily the MikroTik "Address").
  Example: server on 10.20.0.1 and MikroTik on 10.20.0.2 → register in `nas` with the same secret and nasname = 10.20.0.2.
  If radius.log shows: Ignoring request ... unknown client <ip>, add that IP to `nas`, then:
  docker compose restart freeradius

Notes
-----
- User auth from radcheck / radreply; NAS rows from `nas` (read_clients = yes).
- authorize_check_query in queries.conf reads users from radcheck; nasname and secret come from `nas` via client_query, not authorize_check_query.
- If `server` column is missing in nas for external schema: Docker image patches client_query in mods-available/sql.template (NULL AS server) after merging default queries.conf.

- Access denied for user 'radius'@'...': user not created or password mismatch vs RADIUS_DB_PASSWORD in .env (same value as freeradius MYSQL_PASSWORD).
  On first MySQL init, docker/mysql/init/z-futureradius-followup.sh may run; you can fix with:
  ALTER USER 'radius'@'%' IDENTIFIED BY '...'; GRANT ALL ON radius.* TO 'radius'@'%'; FLUSH PRIVILEGES;

- After importing a full radius.sql dump: MySQL grants for the FreeRADIUS user may be missing.
  Align the radius user password with RADIUS_DB_PASSWORD, then restart freeradius. Otherwise you may see high CPU / Docker appearing hung.

radacct indexes
---------------
  Apply sql/migrations/003_radacct_indexes.sql once on the radius database.

Simultaneous-Use (concurrent session limit)
------------------------------------------
  Policy file policy.d/futureradius-simultaneous-use calls futureradius_check_simultaneous_use after the sql authorize step.
  After Dockerfile changes: docker compose build freeradius && docker compose up -d freeradius

  Projects with legacy_cards: store simultaneous_use on cards; see sql/migrations/998-rm-cards-simultaneous-use.sql
  (numeric migrations run when legacy_users exists).

Troubleshooting: "exec /docker-entrypoint.sh: exec format error"
----------------------------------------------------------------
  Usually CPU architecture mismatch (e.g. arm64 image on amd64 host) or CRLF line endings on *.sh.
  Fix: rebuild after pulling latest — Dockerfile uses FROM --platform=linux/amd64 and compose sets
  platform: linux/amd64 for freeradius. Re-normalize line endings: git add --renormalize docker/freeradius/*.sh
  Then: docker compose build --no-cache freeradius && docker compose up -d freeradius
